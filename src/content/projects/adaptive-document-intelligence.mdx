Financial documents are messy — inconsistent layouts, varied formats, tables buried in prose. This project is an attempt to extract structured information from them reliably.

The pipeline is modular: each stage handles validation, testing, or document processing independently and can be swapped without breaking the rest. The goal was to build something that felt like a production codebase — with documentation, changelog, evaluation reports, and containerised execution — not a notebook experiment.

The feedback-driven loop is what makes it adaptive: extraction errors feed back into the system to improve subsequent runs without retraining the base model.

**What I'd improve next:** better handling of edge-case document layouts, and a cleaner evaluation harness that separates field-level accuracy from document-level accuracy.
